# Week 01 Solutions

## 📋 Overview
This folder contains solutions for Week 01 assignments as part of the CTS Digital Nurture 5.0 program.

## 📁 Files

| File | Description |
|------|-------------|
| 01)SingletonPatternExample | Singleton Design Pattern – ensures only one instance of a class is created |
| 02)FactoryMethodPatternExample | Factory Method Design Pattern – creates objects without specifying the exact class |
| 03)EcommercePlatformSearch | Search algorithms (Linear & Binary Search) for an e-commerce product catalog |
| 04)FinancialForecasting | Recursive algorithm to calculate future financial values |
| 05)Control Structures | PL/SQL Control Structures – IF-ELSE, LOOP, WHILE, FOR loop examples |
| 06)Stored Procedures | PL/SQL Stored Procedures – creating and executing stored procedures in Oracle XE |
| 07)Setting up JUnit | JUnit 4 setup with Maven in VS Code – unit testing configuration |
| 08)Assertions in JUnit | JUnit 4 assertions – assertEquals, assertTrue, assertFalse, assertNull, assertNotNull |
| 09)AAA Pattern and Setup Teardown | Arrange-Act-Assert pattern with @Before and @After lifecycle methods |
| 10)Mocking and Stubbing | Mockito mocking and stubbing – testing services with mocked external dependencies |
| 11)Verifying Interactions | Verifying method calls on mocks using Mockito's verify() |
| 12)Logging Error Messages and Warning Levels | Logging with different severity levels – ERROR, WARN, INFO using a logging framework |

## 🔍 Notes
- Built using Java (JDK 17), Maven, and VS Code
- PL/SQL exercises run on Oracle Database XE (local)
- JUnit tests configured with `maven-archetype-quickstart`
- Mockito used with JUnit 4 for mocking and verifying interactions
